# 测试生成流水线 — 穷尽式优化分析（22+14维，目标100+点）

## 1. 功能架构 (Architecture)

| # | 优化点 | 严重度 | 说明 |
|---|--------|--------|------|
| 1 | **人工审查Gate缺失** | 🔴 关键 | 生成→运行→报告后无人工确认环节，应有"预览→接受/拒绝/修改→提交"的人机协作闭环 |
| 2 | **A/B测试模式** | 🟡 重要 | 同时用qodo-cover(LLM)和EvoSuite(搜索)生成同一函数测试，自动选质量更优者 |
| 3 | **增量测试生成** | 🔴 关键 | 代码diff后应只重新生成受影响模块的测试，当前是全量分析模式 |
| 4 | **测试意图标注** | 🟡 重要 | 每个生成的测试应标注测试目标(正常路径/边界/异常/并发)，便于审查和维护 |
| 5 | **测试去重引擎** | 🟡 重要 | LLM可能对同一逻辑生成语义重复的测试，需AST级别的等价性检测 |
| 6 | **Dry-Run模式** | 🟢 增强 | 只分析不生成，输出"将生成X个测试，预估覆盖率Y%"，用于CI快速门禁 |
| 7 | **统一AST中间表示(IR)** | 🔴 关键 | 每种语言独立解析增加N倍维护成本，需要tree-sitter统一IR层 |
| 8 | **测试优先级分级** | 🟡 重要 | P0(关键路径) > P1(边界条件) > P2(异常路径) > P3(组合覆盖)，可按级执行 |
| 9 | **依赖感知生成** | 🔴 关键 | 修改模块A → 应同时检查/更新依赖A的模块B的测试 |
| 10 | **测试模板库** | 🟡 重要 | 常见模式(CRUD/认证/分页/文件上传)预置测试模板，非LLM路径直接填充 |
| 11 | **策略热插拔** | 🟡 重要 | 新测试生成策略可通过插件注册，不修改核心代码 |

## 4. 可靠性 (Reliability)

| # | 优化点 | 严重度 | 说明 |
|---|--------|--------|------|
| 28 | **LLM→模板Fallback** | 🔴 关键 | LLM不可用时自动降级到模板生成, 确保流水线不中断 |
| 29 | **Docker→本地Fallback** | 🟡 重要 | Docker不可用时降级到本地执行(带警告), 而非直接失败 |
| 30 | **覆盖率工具不可用降级** | 🟡 重要 | 覆盖率收集失败时跳过门禁继续生成, 标记结果降级 |

| 32 | **网络不稳定缓存** | 🟡 重要 | LLM API调用失败时使用本地缓存的相似场景响应 |
| 33 | **幂等性保证** | 🔴 关键 | 同一输入多次运行产生相同结果, 支持 CI 重复执行 |
| 34 | **超时分级控制** | 🟡 重要 | Lint:30s / LLM单次:120s / 测试执行:300s / 全流程:1800s |
| 35 | **资源泄漏检测** | 🟢 增强 | 测试执行后自动检测未关闭的文件/连接/进程, 清理+报告 |
